## Description

A build refers to a specific build of a software.  It tracks to progress of building the software docker image and is associated with runs that use it.

## Commands

### Find
```shell
$ carrot_cli software version build find --help
Usage: carrot_cli software version build find [OPTIONS]

  Retrieve software build records filtered to match the specified parameters

Options:
  --software_build_id TEXT    The ID of the software build record, a version 4
                              UUID

  --software_version_id TEXT  The ID of the software version to find build
                              records of

  --build_job_id TEXT         The cromwell job id for the job that built that
                              generated the build

  --status TEXT               The status of the build job. Statuses include:
                              aborted, created, expired, failed,
                              queued_in_cromwell, running, starting,
                              submitted, succeeded, and
                              waiting_for_queue_space

  --image_url TEXT            The URL indicating the location where the Docker
                              image generated by the build is hosted

  --created_before TEXT       Upper bound for software build's created_at
                              value, in the format YYYY-MM-DDThh:mm:ss.ssssss

  --created_after TEXT        Lower bound for software build's created_at
                              value, in the format YYYY-MM-DDThh:mm:ss.ssssss

  --finished_before TEXT      Upper bound for software build's finished_at
                              value, in the format YYYY-MM-DDThh:mm:ss.ssssss

  --finished_after TEXT       Lower bound for software build's finished_at
                              value, in the format YYYY-MM-DDThh:mm:ss.ssssss

  --sort TEXT                 A comma-separated list of sort keys, enclosed in
                              asc() for ascending or desc() for descending.
                              Ex. asc(software_name),desc(created_at)

  --limit INTEGER             The maximum number of software build records to
                              return  [default: 20]

  --offset INTEGER            The offset to start at within the list of
                              records to return.  Ex. Sorting by
                              asc(created_at) with offset=1 would return
                              records sorted by when they were created
                              starting from the second record to be created
                              [default: 0]

  --help                      Show this message and exit.
```

### Find by id
```shell
$ carrot_cli software version build find_by_id --help
Usage: carrot_cli software version build find_by_id [OPTIONS] ID

  Retrieve a software build record by its ID

Options:
  --help  Show this message and exit.

```
